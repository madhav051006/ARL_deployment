# ACIDS DeepSense C Inference Package (n_fft=25)

Self-contained C inference package for the W8-quantized DeepSense audio model trained with **`n_fft=25`** mel.

This git repository **is** the deploy package: clone it and `Makefile`, `main.c`, `samples/`, etc. live at the **repo root** (no nested project folder).

Remote: `git@github.com:madhav051006/ARL_deploy.git`

## Build And Run

```bash
git clone git@github.com:madhav051006/ARL_deploy.git
cd ARL_deploy
make
./acids_infer
```

If you already have this tree (e.g. `demo_codebase/ARL_deploy/`), from the repo root:

```bash
make
./acids_infer
```

Running `./acids_infer` with no arguments runs all 15 packaged samples in `samples/`.

To run one explicit sample:

```bash
./acids_infer samples/Gv3c1090_96.bin
```

Convert a human-readable `.txt` sample to `.bin` (no Python needed):

```bash
./acids_txt_to_bin samples_txt/Gv3c1090_96.txt /tmp/Gv3c1090_96.bin
./acids_infer /tmp/Gv3c1090_96.bin
```

Regenerate packaged samples from ACIDS `.pt` files (Python, requires repo `src2/`):

```bash
python3 export_c_raw_samples.py
```

Verify C mel preprocessing matches PyTorch:

```bash
python3 test_mel_parity_nfft25.py
```

## Input Format

Each sample is raw float32 binary with this flattened shape:

```text
[C=3][segments=7][T=25]
```

This is **decimated audio at 1600 Hz** (25 samples per segment). It matches the Python ACIDS pipeline after `AudioDownsampler(16000 -> 1600)`:

```text
audio: (3, 7, 25) @ 1600 Hz per segment
```

| Field | Value |
|-------|-------|
| Dtype | `float32` |
| Layout | CHW when flattened |
| Total values | 525 |
| File size | 2,100 bytes (525 × 4) |

## Clip Duration (important)

This model sees a **short ACIDS event window (~110 ms)**.

| Stage | Shape | Sample rate | Duration |
|-------|-------|-------------|----------|
| ACIDS .pt (native) | (3, 7, 256) | 16 kHz | ~112 ms |
| After decimation | (3, 7, 25) | 1600 Hz | ~109 ms |
| After mel | (1, 7, 80) | — | 7 time bins |

Flatten order (channel-major):

```text
index(c, seg, t) = c * (7 * 25) + seg * 25 + t
```

The C runner uses only **audio channel 0**. Channels 1 and 2 are ignored.

Your upstream pipeline must deliver audio already decimated to 1600 Hz with 25 samples per segment. The C code does **not** perform 16 kHz → 1600 Hz downsampling.

### Human-readable `.txt` format

The same data is also available in `samples_txt/` (one float per line, with a short header):

| Line | Content | Example |
|------|---------|---------|
| 1 | Sample name | `Gv3c1090_96` |
| 2 | Label id (use `-1` if unknown) | `0` |
| 3 | Layout tag (fixed) | `RAW_AUDIO_CHW` |
| 4 | Shape (fixed for this model) | `3,7,25` |
| 5–529 | Flattened CHW values | one float per line |

Convert `.txt` → `.bin` with `./acids_txt_to_bin` (built by `make`).

### Example Python writer

```python
import numpy as np

NUM_CHANNELS = 3
NUM_SEGMENTS = 7
SAMPLES_PER_SEGMENT = 25


def write_sample_bin(path: str, audio_chw: np.ndarray) -> None:
    if audio_chw.shape != (NUM_CHANNELS, NUM_SEGMENTS, SAMPLES_PER_SEGMENT):
        raise ValueError(f"expected (3, 7, 25), got {audio_chw.shape}")
    audio_chw.astype(np.float32).tofile(path)


def write_sample_txt(path: str, sample_name: str, label_id: int, audio_chw: np.ndarray) -> None:
    flat = audio_chw.astype(np.float32).reshape(-1)
    with open(path, "w", encoding="utf-8") as f:
        f.write(f"{sample_name}\n{int(label_id)}\nRAW_AUDIO_CHW\n3,7,25\n")
        for value in flat:
            f.write(f"{float(value):.9g}\n")
```

## Data Flow

```text
sample .bin  [3][7][25] @ 1600 Hz
  -> main.c
  -> acids_audio_preprocess_ch0_segments_chw()
       select channel 0
       log-mel per segment (n_fft=25, no padding)
  -> model_forward()
  -> argmax logits
```

## Main Files

`main.c`

- Opens one or more raw sample binaries.
- If no input path is provided, loops through packaged samples from `acids_samples.h`.
- Calls `acids_audio_preprocess_ch0_segments_chw()`.
- Calls `model_forward()`.
- Prints expected class, predicted class, logits, and match status.

`acids_samples.h`

- Lists packaged sample names, paths, expected label ids, and expected label names.

`acids_class_names.h`

- Maps model output indices to class names:

```text
0 -> background
1 -> 3
2 -> 6
3 -> 8
4 -> 9
```

`acids_mel_preprocess.c/.h`

- Implements C preprocessing before model inference (n_fft=25 mel).

`model.c/.h`, `weights.h`

- Generated by Tiny-NN-in-C from the finetune checkpoint below.
- `model_forward(const float *input, float *output)` expects post-mel model input.

`nn_ops_float.h`, `nn_ops_int8.h`, `nn_ops_int16.h`

- Runtime operators used by the generated model.

## Preprocessing Details

Entry point:

```c
int acids_audio_preprocess_ch0_segments_chw(const float *input_chw, float *out_chw);
```

Input:

```text
input_chw: [3][7][25] float32, flattened CHW @ 1600 Hz per segment
```

Output:

```text
out_chw: [1][7][80] float32, flattened CHW
```

### Channel Selection

The function selects channel 0:

```text
input channel 0 -> [7][25]
```

Channels 1 and 2 are ignored.

### Mel Preprocessing

Each segment has 25 samples. With `n_fft=25`, **no zero-padding** is applied before the RFFT.

Mel parameters (matches `finetune_audio_deepsense_dw_large_mel_5class_nfft25`):

```text
n_fft: 25
mel_bins: 80
fmin: 20.0 Hz
fmax: 800.0 Hz
sample_rate: 1600 Hz
log epsilon: 1e-6
```

Pipeline per segment:

```text
25-sample segment @ 1600 Hz
  -> RFFT, 13 bins
  -> power spectrum
  -> mel filterbank [80][13]
  -> log(mel + 1e-6)
```

Lookup tables:

- `acids_rfft_tables.h`: RFFT basis matching `torch.fft.rfft` with n_fft=25.
- `acids_mel_tables.h`: mel filterbank matching Python `MelPreprocessor`.

Regenerate tables after mel param changes:

```bash
python3 ../src2/inference_c/export_mel_tables.py \
  --n_fft 25 --n_mel 80 --fmin 20 --fmax 800 --sample_rate 1600 \
  --output acids_mel_tables.h \
  --rfft_output acids_rfft_tables.h
```

## Model Input And Output

Model input after preprocessing:

```text
[1][7][80] float32
```

Model output:

```text
logits[5]
```

The runner uses argmax over these 5 logits.

| Field | Value |
|-------|-------|
| Experiment | `finetune_audio_deepsense_dw_large_mel_5class_nfft25` |
| Model YAML name | `student_audio_deepsense_dw_large_mel` |
| Architecture | `DeepSenseDepthwiseBackbone` (GELU, 1-channel deploy) |
| Classes | 5 (background, 3, 6, 8, 9) |
| Checkpoint | `src2/experiments/20260706_183051_finetune_finetune_audio_deepsense_dw_large_mel_5class_nfft25/models/best_model.pth` |
| Val accuracy | 69.1% (float32, epoch 8) |
| Compiler | [Tiny-NN-in-C](https://github.com/) (`~/Tiny-NN-in-C`) |
| Export script | `export_w8_model.py` (in this folder) |

Training used 3-channel mel; C deployment uses channel 0 only. The export script slices first-layer conv weights accordingly.

## Quantization

This package uses:

```text
Convolution layers: static int8 per-channel (weights + activation scales)
Linear/projection layers: weight-only int8 per-channel
Activations around Linear layers: float32
```

Activation scales for conv layers were calibrated on 32 ACIDS train batches (n_fft=25 mel, channel 0).

## Regenerate Model Weights

From this directory:

```bash
python3 export_w8_model.py --max_calib_batches 32
```

Then rebuild:

```bash
make clean && make
```

Weights from the old `n_fft=160` package (`deepsense_tiny_c_static_int8_conv_weight_only_linear_deploy`) are **not compatible** with this preprocessing.
